## 对象优化
### 临时对象优化
- **RVO/NRVO**：编译器自动消除临时对象，直接在调用者栈帧构造返回值（C++17 起强制要求）。
- **拷贝省略**：特定场景下（如返回局部变量）跳过拷贝/移动操作。

### 三条优化规则
1. 避免不必要的拷贝（优先用引用/移动语义）。
2. 减少临时对象创建（利用返回值优化）。
3. 依赖编译器优化（如 RVO 的触发条件）。

### 移动语义
- 右值引用（`T&&`）实现资源“窃取”，避免深拷贝。
- 移动构造函数/赋值运算符需置空源对象指针（如 `other.data_ = nullptr`）。

### 完美转发
- `std::forward<T>` 保持参数的值类别（左值/右值），实现泛型转发。

## 智能指针
### 裸指针问题
- 手动管理内存易泄漏，异常不安全。

### 非所有权指针（如 `ScopedPtr`）
- 生命周期与作用域绑定，析构时自动释放资源。
- 禁用拷贝操作（防止重复释放）。

### 共享所有权（`std::shared_ptr`）
- 引用计数管理资源，最后一个持有者释放内存。
- **循环引用问题**：用 `std::weak_ptr` 打破循环。
- **线程安全**：引用计数的增减需原子操作。

### 自定义删除器
- 适配非 `new/delete` 的资源（如文件句柄、网络连接）。

## C++11 总结与多线程
### 核心新特性
- **线程支持**：`std::thread`、`std::async`。
- **同步原语**：`mutex`、`condition_variable`、`atomic`。
- **并发工具**：`future/promise`、`packaged_task`。

### 线程安全通信
- **生产者-消费者**：用条件变量 + 互斥锁同步队列。
- **Future 机制**：异步任务结果传递（`std::future`）。

### 多线程问题
- **竞态条件**：用互斥锁保护共享数据。
- **死锁**：避免嵌套锁或按固定顺序获取锁。

## 设计模式
### 创建型模式
- **单例**：延迟初始化（局部静态变量）、线程安全（C++11 起自动保证）。
- **工厂**：解耦对象创建与使用，支持扩展新类型。

### 结构型模式
- **代理**：控制对对象的访问（如权限检查、延迟加载）。
- **装饰器**：动态添加职责（如 IO 流的多层缓冲）。
- **适配器**：统一接口（如将 `new[]` 适配为 `std::vector`）。

### 行为型模式
- **观察者**：事件驱动（如 GUI 按钮点击通知）。

## STL emplace 方法剖析
### 与 `push_back` 的区别
- **`emplace_back`**：直接在容器内构造对象（参数完美转发）。
- **`push_back`**：需先构造临时对象，再移动/拷贝到容器。

### 适用场景
- 需就地构造时（如插入 `std::pair` 到 `std::map`）。
- 避免不必要的拷贝（尤其对不可拷贝但可移动的类型）。
